# ダブルス・チーム編成／オールエベンツ合算成績

## 管理画面

- `/tournaments/{id}/aggregate-results`: 編成、合算定義、合算元、計算結果を管理する。
- 大会編集画面の「編成・合算成績」から移動できる。
- ダブルス／チーム大会では、`tournament_participants` から組またはチームを編成する。
- オールエベンツでは、シングルス、ダブルス、チーム等を別の `tournaments` として登録し、合算元に指定する。

## DB正本

- `tournament_competitor_groups`: 組／チームの名称、部門、必要人数。
- `tournament_competitor_group_members`: 編成に所属する大会参加者。
- `tournament_aggregate_definitions`: 個人／編成の合算方式、性別、同ピン方針、公開対象。
- `tournament_aggregate_sources`: 合算元大会、ステージ、ゲーム範囲、予定ゲーム数。
- 計算結果は `tournament_result_snapshots` / `tournament_result_snapshot_rows` へ版として保存する。

## 計算ルール

- 元スコアの正本は既存の `game_scores` とし、合算用に複製しない。
- プロは `pro_bowler_id`、アマは `amateur_bowler_id` を優先して大会間を照合する。
- IDがない旧データはライセンス番号または氏名で仮照合し、診断件数を画面へ表示する。
- 組／チームは参加者IDを最優先にスコアへ結び、編成人数と全メンバーのゲーム数を検査する。
- 必須競技の欠落、予定ゲーム数の不足／超過、未編成スコアは結果画面で判別できる。
- 同じスコア行が複数の合算条件へ重複した場合は計算を中止する。
- 同ピン時は大会ごとに「同順位」または「ローハイ差」を選ぶ。既定は同順位。
- 未完了行は参考値として表示するが、管理画面の順位は `-` とする。

## 再計算と履歴

- 「合算を再計算」で新しいスナップショットを作成する。
- 同じ合算定義の旧スナップショットは `is_current=false` にし、履歴を消さない。
- 合算定義や編成を削除しても、過去スナップショットの表示名と内訳は保持する。
- 現段階の合算スナップショットは既存個人成績・タイトルへ自動同期せず、誤反映を防ぐ。
