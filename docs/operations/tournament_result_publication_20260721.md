# 公式大会結果の確定・公開

## 目的

速報入力や各ラウンドの正式成績スナップショットを、管理者確認なしでポイント、賞金、タイトル、ランキング、公開PDFへ反映しないための運用です。

## 基本手順

1. 大会設定でポイント配分、賞金配分、競技方式、タイトル区分を確認する。
2. 速報入力から予選、準決勝、決勝など必要な正式成績スナップショットを作成する。
3. 大会成績の「公式結果の確定・公開」を開く。
4. 最終成績スナップショットを選び、全選手、ポイント内訳、賞金、公開元成績表、警告を確認する。
5. 管理者が確認チェックを入れ、「公式結果として確定する」を実行する。
6. 公開PDF、成績一覧、プロフィールのタイトル、年度ランキング集計を確認する。

## 確定時の処理

- 決勝、ラウンドロビン、準決勝、準々決勝、予選の現行スナップショットを順位付きで統合する。
- 同一選手は最上位ステージの行を採用し、敗退者を後続ステージから補完する。
- プロはライセンス番号を選手マスタへ解決し、アマチュアは公式ポイントと賞金を0にする。
- 通常大会は登録済みポイント／賞金配分を適用する。
- シーズントライアルは1位から8位の入賞ポイントと、準決勝順位に基づくステップポイントを合算する。
- `tournament_results` を確定内容へ置換し、ポイント、賞金、アベレージ、公開成績一覧の正本とする。
- タイトル対象大会は1位選手のタイトル明細を同期し、対象外大会や予選は除外する。
- 公開対象スナップショットだけを `is_published=true` にし、PDFの再出力元を固定する。
- 確定内容を公開履歴と明細へ複製し、チェックサム、実行者、日時、反映件数を保存する。

## 公開を止める検査

- 最終成績でない、旧版である、シフト別だけの成績である。
- 大会と成績表の性別が一致しない。
- 男女別大会で、男子または女子の片方だけを大会全体の結果として選択している。
- 未入力行、同一成績内の選手重複、プロ選手マスタへ解決できない行がある。
- チーム／ダブルスの合算成績に個人ポイント・賞金の配分規則がない。
- 最終確認後に成績または配分が変わり、チェックサムが一致しない。
- ポイント／賞金対象なのに、配分表も正の保存値もない。初期値0だけでは設定済みと判定しない。

## 訂正と改訂

- 公開済みの `tournament_results` は直接編集・削除しない。
- 元の速報、スコア、スナップショットを訂正して再度プレビューする。
- 管理者が再確定すると旧公開版は `superseded`、新公開版は `current` となる。
- 公開履歴明細は当時値を保持し、後から選手名や所属が変わっても確定時の内容を確認できる。
- 同順位は同じ順位のまま保持し、合算成績の競技別内訳も公開履歴へ引き継ぐ。

## 権限

- 編集者と管理者はプレビューを閲覧できる。
- 確定公開は管理者だけが実行できる。
- 公開後の通常成績追加、編集、削除は画面とサーバーの両方で拒否する。
