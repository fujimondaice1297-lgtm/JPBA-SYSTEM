# スコア取込運用手順書

最終更新: 2026-06-26

## 目的

CSV、Excel/Googleスプレッドシート貼り付け、OCR JSON、OCR/AI出力貼り付けを、すべて同じ確認用ステージング運用にそろえる。

重要な原則:

- 取込結果は直接 `game_scores` へ入れない。
- まず `score_import_batches` / `score_import_rows` に保存する。
- 要確認行を人間が確認・修正する。
- 確認済み行だけを `game_scores` へ確定反映する。
- OCR結果は正本ではない。正本は確認後に `game_scores` へ入ったスコア。

## 画面の入口

大会ごとの運用ログ画面を開く。

- 画面: `tournaments/{tournament}/operation-logs`
- スコア取込欄: `スコア一時取込`
- 取込後の確認画面: `tournaments/{tournament}/score-imports/{scoreImport}`

## 取込方式ごとの使い分け

### 1. CSVファイル取込

使う場面:

- 成績表をCSVとして保存できる。
- 表の列名がある程度そろっている。
- 1行1ゲーム形式、または `1G` / `2G` / `3G` の横持ち形式で渡せる。

対応しやすい列名:

- `license_number`
- `license_no`
- `ライセンス番号`
- `name`
- `氏名`
- `entry_number`
- `stage`
- `game_number`
- `score`
- `shift`
- `gender`
- `1G` / `2G` / `3G`
- `G1` / `G2` / `G3`
- `第1ゲーム`

手順:

1. 運用ログ画面の `CSVファイル` へファイルを選ぶ。
2. 必要なら既定ステージ、既定G、既定シフト、既定性別を入力する。
3. `CSVを一時取込` を押す。
4. 取込一覧の `詳細` を開く。
5. 要確認行を修正する。
6. `確認済み行を反映` を押す。

### 2. Excel/Googleスプレッドシート貼り付け

使う場面:

- 現場でExcelやGoogleスプレッドシートを使っている。
- ファイルアップロードよりコピー貼り付けのほうが早い。
- 表の列を人間が見ながら整えられる。

推奨フォーマット:

```text
license_number	name	stage	1G	2G	3G
M00001297	山田太郎	予選	210	225	198
F00000001	鈴木花子	予選	200	216	190
```

手順:

1. 表計算ソフトでヘッダー行を含めてコピーする。
2. 運用ログ画面の `Excel貼り付け` へ貼り付ける。
3. 必要なら既定ステージ、既定G、既定シフト、既定性別を入力する。
4. `貼り付けを一時取込` を押す。
5. 詳細画面で要確認行を修正する。
6. `確認済み行を反映` を押す。

### 3. 写真/PDF原本の保存

使う場面:

- 紙で提出された成績表を写真やPDFで保存する。
- まだOCR解析はしないが、原本を先にシステムへ残したい。
- 後からOCR JSONやOCR/AI出力貼り付けを追加したい。

手順:

1. 運用ログ画面の `写真/PDF原本` へ画像またはPDFを選ぶ。
2. 既定ステージ、既定シフト、既定性別、メモを入力する。
3. `写真/PDFを保存` を押す。
4. 取込一覧に `写真/PDF` のバッチができる。
5. 詳細画面を開き、OCR解析結果を追加する。

注意:

- この段階では `score_import_rows` は作られない。
- 状態はOCR解析待ちの原本として残る。
- 後続のOCR結果は同じバッチに追加する。

### 4. OCR JSON取込

使う場面:

- 外部OCRやAI処理がJSONを返せる。
- `rows` / `players` / `results` / `items` のような配列で結果を持てる。
- 日本語キーを含むJSONを使いたい。

対応しやすいキー:

- `license_number`
- `ライセンス番号`
- `name`
- `氏名`
- `stage`
- `ステージ`
- `game_number`
- `ゲーム番号`
- `score`
- `スコア`
- `confidence`
- `信頼度`
- `scores`
- `games`

例:

```json
{
  "rows": [
    {
      "ライセンス番号": "M00001297",
      "氏名": "山田太郎",
      "ステージ": "予選",
      "scores": {
        "1": 210,
        "2": 225,
        "3": 198
      },
      "信頼度": 92
    }
  ]
}
```

手順:

1. 写真/PDF原本バッチの詳細画面を開く。
2. `OCR解析結果JSON取込` にJSONファイルを選ぶ。
3. 必要なら既定ステージ、既定シフト、既定性別を入力する。
4. 既存解析行があり、差し替えたい場合だけ `既存解析行を差し替える` を選ぶ。
5. `JSONを取込` を押す。
6. 行一覧で信頼度、要確認理由、変換元を確認する。
7. 要確認行を修正して `確認済み行を反映` を押す。

### 5. OCR/AI出力貼り付け

使う場面:

- 外部OCRやAIから、表テキストやMarkdown表が返ってくる。
- JSONにする前のテキストを一度画面で確認したい。
- 実OCRエンジン接続前に、人間が貼り付けて動作確認したい。

対応形式:

- JSON
- Markdown表
- タブ区切り表
- カンマ区切り表
- 空白区切りの簡易表

手順:

1. 写真/PDF原本バッチの詳細画面を開く。
2. `OCR/AI出力貼り付け` にテキストを貼る。
3. 必要なら既定ステージ、既定シフト、既定性別、既定Gを入力する。
4. まず `変換JSONを確認` を押す。
5. 別タブで `summary`、`boundary`、`payload.rows` を確認する。
6. 問題なければ元画面へ戻り `貼り付け変換` を押す。
7. 行一覧で要確認行を修正する。
8. `確認済み行を反映` を押す。

## 実OCRエンジン接続時の境界

実OCRエンジンを接続する場合も、既存の確認経路を変えない。

固定する流れ:

```text
画像/PDF原本バッチ
  -> OCR処理
  -> ScoreImportOcrEngineBoundaryService
  -> ScoreImportOcrTextAdapterService
  -> ScoreImportOcrResultStageService
  -> score_import_rows
  -> 人間確認
  -> game_scores
```

実装上の入口:

- OCRエンジンへ渡す入力仕様: `ScoreImportOcrEngineBoundaryService::buildEngineInput()`
- OCR/AIテキストを確認用行へ変換: `ScoreImportOcrEngineBoundaryService::stageTextResult()`
- 変換だけ確認: `ScoreImportOcrEngineBoundaryService::previewTextResult()`

禁止事項:

- OCRエンジンから `game_scores` へ直接INSERT/UPDATEしない。
- OCRエンジンから `tournament_results` へ直接INSERT/UPDATEしない。
- OCR結果だけで順位、タイトル、PDFを自動更新しない。

## 詳細画面で確認する項目

取込詳細画面では以下を確認する。

- 全行数
- 自動確認済み
- 手動確認済み
- 要確認
- 除外
- 反映済み
- OCR/AI変換サマリー
- 行ごとの信頼度
- 要確認理由
- 変換元ファイル/行/列
- 候補選手
- ステージ/G/スコア/シフト/性別

要確認理由の例:

- `stage_missing`: ステージ未入力
- `game_number_missing`: G未入力
- `score_missing`: スコア未入力
- `score_out_of_range`: スコア範囲外
- `player_identity_missing`: 選手識別なし
- `player_unmatched`: 選手未照合
- `player_ambiguous`: 候補複数

## 修正と確定反映

個別修正:

1. 行の候補、参加者ID、プロID、ライセンス、氏名、ステージ、G、スコアを確認する。
2. 必要な項目を修正する。
3. 状態を `確認済み` または `要確認` または `除外` にする。
4. `保存` を押す。

一括修正:

1. 対象行にチェックを入れる。
2. ステージ、G、シフト、性別などを入力する。
3. 必要に応じて `空欄だけ` を使う。
4. `選択行に適用` を押す。

確定反映:

1. 要確認行が残っていないか確認する。
2. 除外すべき行は `除外` にする。
3. `確認済み行を反映` を押す。
4. 反映後、`反映済み: game_scores #...` が付く。
5. 操作ログで件数とメッセージを確認する。

## 差し替えと再取込

反映前:

- 既存解析行を差し替えられる。
- `既存解析行を差し替える` は、OCR JSONやOCR/AI貼り付けのやり直しに使う。

反映後:

- 反映済み行があるバッチは、解析行の差し替えをしない。
- 誤反映があった場合は、別の修正手順を作る。安易にバッチを消さない。

## トラブル時

CSVのヘッダーが読めない:

- 1行目に列名があるか確認する。
- Shift-JIS/CP932のCSVで保存し直す。
- Excel貼り付けへ切り替える。

選手が照合されない:

- ライセンス番号の桁、接頭文字、空白を確認する。
- 氏名のスペースや旧字体を確認する。
- 候補が出ていれば手動で選ぶ。

スコアが読み取れない:

- `score` 列、または `1G` / `2G` / `3G` 形式になっているか確認する。
- OCR出力の場合は `変換JSONを確認` で `payload.rows` を見る。

候補が複数出る:

- 候補選択欄から正しい選手を選ぶ。
- 同姓同名やライセンス欠落の場合は、ライセンス番号を補う。

## 完了条件

この運用が完了したと言える状態:

- 取込バッチが作成されている。
- 取込行が `score_import_rows` にある。
- 要確認行が修正または除外されている。
- 確認済み行が `game_scores` へ反映されている。
- 操作ログに取込、修正、確定反映の履歴が残っている。
- 大会終了チェックリストでスコア不足や未反映候補が残っていない。
