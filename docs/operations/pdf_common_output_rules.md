# 大会PDF共通表示ルール

最終更新: 2026-06-29

## 目的

大会PDFは公式帳票に近い見た目を保ちつつ、方式別の専用表示が他方式へ漏れないようにする。この文書を大会PDF表示の共通ルール正本とする。

## 共通ルール

| 項目 | ルール |
|---|---|
| 氏名 | 公式PDFに寄せ、姓と名の間は全角スペースを基本にする。DB上に既に表示用氏名がある場合は、二重スペース化しない。 |
| フリガナ | 表示できる方式では、氏名と同じ選手を指す補助情報として扱う。空欄の場合は無理に推測しない。 |
| 期表示 | プロフィール、snapshot row、方式別seed rowの順に拾える値を使い、表示時は `期生` 表記へ揃える。既に `期` を含む値は重複させない。 |
| ライセンスNo | PDF上は原則として末尾4桁表示。シード判定がある場合は `S 1234` のように `S` を付ける。 |
| アマチュア | ライセンス欄は `アマ` または既存のアマ識別に揃え、プロライセンスのように見せない。 |
| 賞金欄 | 金額は折り返し・枠はみ出しを起こさない幅で表示する。空欄と0円の扱いは大会成績側の既存ルールに合わせる。 |
| AVG | `total_pin / games` を基準に小数2桁へ揃える。ゲーム数0の行は除算しない。 |
| シード表示 | 大会成績一覧、snapshot、速報、PDFで同じシード判定を使う。年度別シードと大会別追加シードの重複表示を避ける。 |
| 方式別文言 | シーズントライアル、シュートアウト、シングルエリミネーション、ステップラダー、通常成績表の文言を互いに混入させない。 |
| 画像生成 | スコアシートやトーナメント図の画像生成でPHPの非推奨警告を出さない。警告が出る状態はPDF回帰NGとする。 |

## 方式別の隔離

| 方式 | 表示責務 |
|---|---|
| 通常成績表 | 最終順位、総ピン、ゲーム数、AVG、ポイント、賞金を中心に表示する。 |
| シーズントライアル | ST専用の進出ライン、ポイント表、シュートアウト図の組み合わせをこの方式内に閉じ込める。 |
| ラウンドロビン | W-L-T、Bonus、RR合計、TOTAL POINT、ステップラダー進出者を表示する。 |
| ステップラダー | テンプレート画像に氏名、スコア、勝者ルートを重ねる方式を維持する。 |
| シュートアウト | seed順、1st/2nd/優勝決定戦、敗退順位、winner note、スコアシートを表示する。 |
| シングルエリミネーション | トーナメント表、勝者線、同順位扱い、レーン表示、`single_elimination_final` を専用表示に閉じ込める。 |

## 回帰確認の最低ライン

1. 対象大会PDFが `%PDF` ヘッダ付きで生成できる。
2. PDF生成中にPHP warning / deprecated warningを出さない。
3. 専用文言が別方式のPDFへ混入しない。
4. 氏名、期、ライセンス下4桁、賞金欄が枠内に収まる。
5. 方式別の図、スコア表、最終順位が消えていない。

2026-06-29時点の実行結果は `docs/operations/result_flow_regression_audit.md` を参照する。
