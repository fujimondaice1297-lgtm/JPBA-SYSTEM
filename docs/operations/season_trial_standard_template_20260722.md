# シーズントライアル標準テンプレート運用（2026-07-22）

## 目的

- 大会ID61の実データを壊さず、2026サマーシリーズの年度開催へ割り当てる。
- 会場ごと・季節ごとに再利用できるシーズントライアル標準テンプレートを作る。
- 選手、エントリー、レーン割当、スコア、成績、タイトルを次大会へコピーしない。

## 公式仕様の根拠

- 2026サマー: https://www.jpba.or.jp/information/tournament/tournament2026/ST_Summer/ST_Summer2026.html
- 2026スプリング: https://www.jpba.or.jp/information/tournament/tournament2026/ST_Spring/StSpring2026.html
- 男子トーナメントプロ（シードを含む）が出場対象で、各季節の成績により後続期間の出場優先順位を決定する。
- 標準進行は予選8G、準決勝4G（通算12G）、上位8名の決勝シュートアウトとする。

## 登録済みID

- シリーズ: ID2 `JPBAシーズントライアル` / code `jpba-season-trial`
- 年度開催: ID2 `JPBAシーズントライアル2026 サマーシリーズ` / season `summer`
- テンプレート: ID2 `シーズントライアル標準（予選8G・準決勝4G・8名シュートアウト）`
- 最新版: v2 / 版ID3
- 適用済み大会: ID61 `メリーランドカップ JPBAシーズントライアル2026 サマーシリーズ B会場`

## 標準設定

| 項目 | 設定 |
|---|---|
| 性別 | 男子 |
| 大会区分 | 公認 |
| 競技単位 | シングルス |
| 年度シード自動登録 | 使用しない |
| JPBA公式ポイント | 対象外 |
| STチャンピオンシップポイント | 対象 |
| 公式アベレージ | 対象 |
| 賞金 | 対象 |
| 後続大会の出場優先順位 | 対象 |
| タイトル反映先 | シーズントライアル優勝 |
| 予選 | 8G |
| 準決勝 | 4G、予選との通算12G |
| 決勝進出 | 通算上位8名 |
| 決勝方式 | 標準8名シュートアウト |

年度シード選手も出場できるが、シーズントライアル内で優先枠として自動登録はしない。公式ポイントランキングには加算せず、ST専用ポイントとして扱う。

## テンプレートに含めないもの

- 開催区分（春夏秋冬は大会作成時に必ず選ぶ）
- 会場ID、名称、住所、TEL、FAX
- 使用レーン範囲、追加会場
- 開催日、大会名、主催・協賛・放送情報
- 参加人数、各段階の通過人数（決勝8名を除く）
- 選手、アマチュア、エントリー、レーン割当
- スコア、成績、スナップショット、タイトル
- ポイント配分表、賞金配分表

ポイント・賞金の具体的配分は、各大会の正式要項を確認して公開前に設定する。

## 実行コマンド

既定はドライランで、DBへ書き込まない。

```powershell
php artisan jpba:setup-season-trial-template 61 `
  --season-key=summer `
  --edition-name="JPBAシーズントライアル2026 サマーシリーズ" `
  --edition-start=2026-07-01 `
  --edition-end=2026-07-28 `
  --edition-status=in_progress `
  --json
```

確認後、`--force` を追加して確定する。

## 保護検査

確定処理は大会行をロックしたトランザクション内で実行し、次の実績データを適用前後で件数・内容チェックサム比較する。

- `tournament_entries`
- `tournament_participants`
- `game_scores`
- `tournament_results`
- `tournament_result_snapshots` / rows
- `tournament_seed_players`
- `tournament_match_score_sheets`
- `tournament_round_lane_assignments`
- `tournament_result_publication_rows`
- `pro_bowler_titles`

大会ID61の最終チェックサムは `5935e70a504781c026f366dbd845a8f351b7ab14a8d2b80342ba78624d144fbe`。再適用時も全変更false、チェックサム一致を確認済み。

## 次の運用

1. 公式大会ページと大会要項を1大会ずつ確保する。
2. シーズントライアルは本テンプレートを適用し、季節、会場、日付、大会名、参加人数、通過人数を入力する。
3. 正式要項に基づいてポイント・賞金配分を登録する。
4. 選手登録、レーン割当、スコア入力、成績確定、PDF表示確認、管理者承認公開の順で進める。
5. 公開後は元成績を直接編集せず、訂正スナップショットから改訂版を再公開する。
