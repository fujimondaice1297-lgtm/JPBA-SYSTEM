# 大会シリーズ・テンプレート運用基盤

## 登録の考え方

1. 継続する大会名を「大会シリーズ」として登録する。
2. 既存大会の設定を「大会テンプレート」として版管理する。
3. 翌年はテンプレートから新大会を作り、日付・大会名・年度固有項目だけを確認する。
4. 同年度の複数会場や男女別競技は、同じ年度開催に属する別の大会として保存する。
5. 選手・スコア・正式成績・タイトル明細は毎大会で新規に管理し、テンプレートから複製しない。

## 管理画面

- `/tournament-templates`: テンプレート一覧。
- `/tournament-templates/create`: 既存大会からテンプレートまたは新版を作成。
- `/tournaments/create`: テンプレートから新大会を作成。
- `/tournaments/{id}/edit`: 集計対象、年度シード、追加優先権、前段階大会、会場を編集。
- `/tournaments/{id}`: 優先出場権を手動再同期できる。

## 自動化される資格

- 当年度シード。大会ごとに使用可否と順位上限を指定する。
- 同じ大会シリーズの歴代公式優勝者。
- 当該年度の別の公式大会優勝者。
- 永久シード保持者。
- 指定した前段階大会の上位N名。

自動同期行には `[AUTO:entry_rule:...]` を付ける。再同期時はこの行だけを更新し、手動で登録した主催者推薦などは維持する。

## 注意事項

- 歴代優勝者の自動抽出には、過去タイトルの `tournament_id` と対象大会の `tournament_series_id` が必要。過去大会をシリーズへ割り当てるほど精度が上がる。
- 前段階大会の通過人数はテンプレートへ引き継ぐが、大会IDは年度固有のため引き継がない。新大会作成時に当年度の予選・順位決定戦を選択する。
- 主催者推薦は人の判断を伴うため、現時点では手動シード登録を使用する。
- ダブルス、チーム、オールエベンツの編成と合算成績は、`tournament_aggregate_results_20260721.md` の実装で対応済み。
- テンプレート版を適用しても、既存大会の選手やスコアは変更しない。
